BMFFFL Data Pipeline
Documentation

How this site gets its data. Written for developers and curious managers who want to understand the architecture behind the stats, standings, and history pages.

Architecture

Data Sources

Sleeper API

Public REST API — no authentication required. Base URL: https://api.sleeper.app/v1

Hardcoded Historical Data

Pre-2020 seasons, all-time records, team names, and award history are stored as static TypeScript data files.

Static Export

Next.js output: 'export' — all pages are pre-rendered HTML. No server runtime.

Status

Current Implementation

The site is fully static. All data is hardcoded. The Sleeper API is available for future dynamic features.

  • Static JSON/TS data files

    Season history, records, rosters

  • No API keys required

    Sleeper API is public

  • Zero server costs

    Deployable to GitHub Pages or Vercel

  • Live score updates

    Not yet implemented

  • Real-time roster sync

    Requires dynamic fetch or ISR

  • Automated transaction feed

    Future integration

Sleeper REST API

Key Endpoints Used

MethodPathDescription
GET/league/{league_id}League settings — scoring, roster limits, playoff format
GET/league/{league_id}/rostersAll roster data — players, picks, taxi, IR for every team
GET/league/{league_id}/matchups/{week}Matchup scores for a given week — points, starters, bench
GET/league/{league_id}/transactions/{round}Trade and waiver history by transaction round
GET/players/nflFull NFL player database — large response, cache aggressively

Base URL: https://api.sleeper.app/v1 — All responses are JSON. No authentication header required.

Sleeper API

Rate Limits

Sleeper does not publish an official rate limit. In practice, keep requests under 5 per second. Burst above that and you may receive transient 429s.

  • No auth token needed

    All league endpoints are public-read

  • Cache /players/nfl

    Large payload (~2 MB) — update weekly at most

  • Batch by week

    Fetch matchups week-by-week, not all at once

  • Be a good citizen

    Sleeper is free — do not hammer their API

Roadmap

Future Integration

What would be needed to make this site data-live — pulling from Sleeper in real time rather than relying on hardcoded exports.

  1. 1

    Replace static JSON exports with server-side Sleeper API calls at build time or via ISR

  2. 2

    Add a Next.js API route (or edge function) to proxy Sleeper requests and avoid CORS

  3. 3

    Cache /players/nfl locally — it is ~2 MB and changes infrequently

  4. 4

    Use NEXT_PUBLIC_SLEEPER_LEAGUE_ID (already in .env) to scope all requests

  5. 5

    Schedule nightly rebuilds or use on-demand revalidation for score updates

How to Contribute

This site belongs to the league. Managers can submit articles, flag data errors, suggest features, and help make BMFFFL the best-documented dynasty league on the internet.

Overview

Ways to Contribute

Submit an Article

Write a Commish Speaks post, analysis piece, or trade retrospective. We publish anything analytically useful or entertainingly Bimfle-adjacent.

See submission guide below

Report a Data Error

Found a wrong score, a missing trade, or a historical record that's off? Flag it and it gets corrected in the next data push.

DM @Grandes on Sleeper

Suggest a Feature

Have an idea for a new analytics page, a poll system, or a trade tool? Open an issue on GitHub or drop it in the Discord.

Open a GitHub issue

Request a Trade Analysis

Want a data-backed breakdown of a trade you're considering or one that already happened? Request one and it may become a published article.

Post in league chat

Articles

Submit an Article

Articles are written in MDX format with a front matter block. The easiest submission paths are a GitHub PR or a Discord message with the file attached.

Required Front Matter

---
title: "Your Article Title"
date: "2026-03-16"
author: "Your Team Name"
slug: "your-article-slug"
category: "analysis"
---

GitHub PR

Fork spruceotter/BMFFFL, add your .mdx file to /content/articles/, and open a pull request.

Discord

Drop the file in #site-submissions on the BMFFFL Discord. Commissioner will handle the merge.

Data Integrity

Report Data Errors

DM @Grandes on Sleeper or create a GitHub issue at spruceotter/BMFFFL.

What to Include

  • The page or section where you found the error

  • What the data currently says

  • What it should say, with a source if possible

  • The season or week it relates to

Who Reviews It

The Commissioner reviews all data corrections. Historical records that change all-time standings go through a brief confirmation period before publishing.

Writing

Article Style Guide

Format Rules

Voice

Bimfle-adjacent: analytical but fun. Stats matter. Trash talk is welcome.

Length

400–1,200 words. Under 400 is a tweet; over 1,200 needs a strong justification.

Format

MDX. Markdown with optional React components for tables and callouts.

Images

Optional. Drop them in /public/articles/{slug}/ and reference by filename.

Required Front Matter Fields

title

The article headline. Keep it under 80 characters.

date

Publication date in YYYY-MM-DD format.

author

Your team name or display name — how you want to be credited.

slug

URL-safe string. e.g. my-trade-analysis-2026. No spaces.

SLA

Response Time

48hrs

Commissioner Review

Articles, data corrections, and feature requests reviewed within 48 hours. Usually faster.

0.003s

Bimfle Review

Bimfle reviews within 0.003 seconds. His opinions are formed before you finish the sentence.

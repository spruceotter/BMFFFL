System Design

How Voting Would Work

1

Manager Authenticates

Commissioner shares a one-time link tied to each manager's Sleeper account.

2

Votes Once Per Poll

Each manager casts a single, irrevocable vote. No changes after submission.

3

Commissioner Tallies

Results are tallied server-side. Live progress shown only after the poll closes.

4

Results Published

Final results posted to the site and announced in league chat. Binding votes go into the rulebook.

What It Would Take

Implementation Requirements

The current site is fully static. A real voting system requires server infrastructure that does not yet exist.

  • Authentication

    Manager identity must be verified — likely via Sleeper OAuth or commissioner-issued tokens

  • Server-side vote storage

    Votes cannot live in localStorage. A backend (Supabase, PlanetScale, or similar) is required

  • One-vote enforcement

    DB constraint or server check prevents double-voting

  • Poll lifecycle management

    Commissioner controls open/close state; results locked on close

Interested?

If you want to help build this, open a GitHub issue or DM the Commissioner. This is one of the higher-effort features on the roadmap.
